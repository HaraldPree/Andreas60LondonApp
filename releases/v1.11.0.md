# v1.11.0 — Bilder-Auswahl für PDF + ZIP (eigene + geteilte)

**Datum**: 23.05.2026
**Status**: deployed
**Git Tag**: v1.11.0

## Anlass

Harald-Wunsch:
> „weiters würde ich gerne die bilder auswählen, die in das fotobuch
> aber auch in das Zip kommen und zwar aus allen bildern die es gibt"

Bisher exportierten PDF + ZIP automatisch **alle eigenen Fotos** (aus
IndexedDB). Geteilte Fotos aus der Gemeinsamen Galerie konnte man nicht
mit ins Foto-Buch nehmen. Das ändert sich jetzt.

## Was ist neu

### Pro Export-Card ein Toggle „Auch geteilte Fotos einbeziehen"

Beide Export-Karten (PDF-Reise-Tagebuch + Foto-Buch-ZIP) haben oben ein
neues Checkbox-Element:

```
☐ Auch geteilte Fotos einbeziehen
   54 geteilte Fotos verfügbar — aktivieren um sie mit aufzunehmen
```

Nach Aktivierung:
```
☑ Auch geteilte Fotos einbeziehen
   38 eigene + 49 aus der Gemeinsamen Galerie = 87 im PDF
```

(Differenz: 5 eigene wurden geteilt → werden dedupliziert, lokale Variante
hat Vorrang weil sie volle Auflösung im Blob hat.)

Button-Label aktualisiert dynamisch: „PDF generieren (87 Fotos)".

### Architektur

**Neuer Typ** `ExportPhoto` (`src/types/photo.ts`):
```ts
interface ExportPhoto extends PhotoMeta {
  remoteUrl?: string;     // Vercel Blob URL bei geteilten Fotos
  uploaderName?: string;  // Wer hat geteilt (für Caption)
}
```

**Neuer Adapter** `src/lib/exportPhotosAdapter.ts`:
- `fromShared(s, tripSlug)` — SharedPhotoView → ExportPhoto
- `combineForExport(own, shared, tripSlug, includeShared)` — vereint, dedupliziert

**Generator-Erweiterungen** (PDF + ZIP):
- Akzeptieren jetzt `ExportPhoto[]` statt `PhotoMeta[]`
- Neuer `loadBlobForExport(p)` Helper: wenn `remoteUrl` gesetzt → fetch,
  sonst IndexedDB-getFullBlob
- Caption mit Quellen-Hinweis: bei geteilten Fotos wird „(geteilt von Andrea)"
  an die Caption angehängt, ohne Caption „Foto von Andrea"
- ZIP-Filenames bekommen `_geteilt-von-Andrea` Suffix damit Foto-Buch-Software
  klar trennen kann

**UI-Erweiterungen**:
- `PdfBookExportButton` + `PhotoBookExportButton`: neuer `sharedPhotos`
  Prop + `includeShared` State + Checkbox
- `FotosTab`: reicht existing `sharedPhotos` (vom useSharedPhotos-Hook)
  durch zu beiden Export-Buttons

### Performance

Geteilte Fotos werden via `fetch(blobUrl)` heruntergeladen. Pro Foto
~100-500 ms Netzwerk-Zeit. Bei 50 geteilten Fotos: ~10-30 s zusätzliche
Wartezeit. Progress-Anzeige im Export-Button zeigt aktuellen Stand.

## Was funktioniert NICHT (bekannte Limits)

- **Keine granulare Foto-Auswahl** — Toggle ist „alle oder keine geteilten".
  Per-Foto-Auswahl käme in **v1.11.1** (Selection-Sheet wie BulkShareSheet)
- **Keine Format-Sortierung** — User-Wunsch „2 quer untereinander neben
  1 hoch" (Smart-Layout) ist weiterhin offen, bleibt für später
- **Nur sichtbare geteilte Fotos** — User sieht laut `canViewSharedPhoto`
  Logik nur Fotos die er sehen darf (group/celebrant). Andere bleiben außen vor
- **Bei Netzwerk-Fail einzelner Fotos** — die werden übersprungen mit
  Console-Warning, der Export läuft trotzdem durch (besser als kompletter
  Abbruch)
