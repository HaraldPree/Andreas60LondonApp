## Bug 1: Goodbye-Reel-Banner verschwand nach Reise-Ende

**Symptom**: Am Abreise-Tag (22.05.2026) öffnete sich der Reel automatisch
+ Banner war sichtbar. Am Tag danach (23.05.) Banner komplett weg, kein
Wiederansehen möglich.

**Ursache**: `ProgrammTab.tsx` Zeile 83–86 (v1.8.0-Code):

```ts
const isLastDay = useMemo(() => {
  const last = trip.days[trip.days.length - 1];
  return last?.isoDate === todayIso; // EXAKTER Match — sonst false
}, [trip.days, todayIso]);
```

Heute (23.05.) ≠ letzter Tag (22.05.) → `isLastDay = false` → Banner gerendert nicht.
Klassischer Off-by-one-day Bug.

**Fix**: zwei separate Conditions:

```ts
const isActualLastDay = useMemo(() => /* === */, …);   // für Auto-Open
const showReelBanner = useMemo(() => /* <= */, …);     // für Render
```

- Auto-Open beim ersten Besuch am echten Abreise-Tag (unverändert, mit
  localStorage-Schutz)
- Banner dauerhaft sichtbar AB Abreise-Tag + danach
- Banner-Label dynamisch: „Letzter Tag — Abschieds-Reel" (am Tag) →
  „Abschieds-Reel · Wiederansehen" (danach)

## Bug 2: PDF/ZIP-Download öffnet about:blank auf Samsung Internet

**Symptom**: Auf Samsung A53 (Samsung Internet Browser) klickt User auf
„PDF speichern" → öffnet leeren `about:blank`-Tab, keine Datei.

**Ursache**: bekannter Samsung-Internet-Bug: `<a download href="blob:…">`
mit programmatisch erzeugtem Blob-URL wird nicht als Download behandelt,
sondern als Navigation → leerer Tab. iOS Safari hat ähnliches Problem.

**Fix in `PdfBookExportButton.tsx` + `PhotoBookExportButton.tsx`**:

Neuer `handleSave()` als primäre Save-Aktion:
1. Wenn `navigator.share` + `canShare({files: [file]})` verfügbar → **Share-Sheet**
   öffnet sich. Auf Samsung/iOS bietet das „In Dateien speichern" /
   „Quick Share" / WhatsApp / Drive / E-Mail
2. Fallback: klassischer `<a download>` Anchor (Desktop, alte Browser)

UI-Umbau:
- Hauptbutton (groß, navy): „PDF speichern" → ruft `handleSave()`
- Untertitel: „Auf Handy: Teilen-Sheet öffnet sich → ‚In Dateien speichern'
  oder App auswählen"
- Sekundär (klein, cream): „Alternativ: Direkt-Download (Desktop / Chrome)" —
  bleibt als `<a download>` für Desktop-User
- Footer-Hint: „Samsung Internet / iOS Safari: blauen Hauptbutton nehmen"

Gleicher Fix für ZIP-Export-Button.

## Was funktioniert NICHT (bekannte Limits)

- **Wenn `navigator.share` File-Sharing nicht unterstützt** (alte Android-Browser):
  Fallback auf `<a download>` — gleiches Problem wie vorher. Workaround:
  Desktop-Browser nutzen
- **Wenn User im Share-Sheet abbricht**: kein zweiter Versuch automatisch.
  Er muss erneut auf „PDF speichern" tippen
- **Reel-Auto-Open hat sich für alte User schon getriggert**: `reelSeenKey`
  Flag bleibt gesetzt, kein Auto-Open mehr. Das ist gewollt — Reel über
  Banner manuell öffnen
