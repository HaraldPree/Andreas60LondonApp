# v1.4.0 — In-App-Editor Phase 2: „Ab hier Rest des Tages offen"

**Datum**: 19.05.2026
**Status**: deployed
**Git Tag**: v1.4.0
**Bump-Grund**: MINOR — neues User-Feature (Phase 2 des In-App-Editors)

## Was ist neu

Phase 2 aus `todo/in-app-day-editor.md`: User können mit einem Tap
**alle Items ab einem Punkt** als ausgelassen markieren — perfekt
für den „wir sind kaputt, Rest des Tages bleibt offen"-Moment.

### Wie's funktioniert

Im bestehenden Item-Action-Sheet (eingeführt in v1.3.0) gibt es jetzt
einen zusätzlichen Schnell-Action-Button **„⏭ Ab hier Rest des Tages
offen lassen"**:

```
┌──────────────────────────────────────┐
│ Item bearbeiten · 14:45         [X]  │
│ Spaziergang im Hyde Park             │
├──────────────────────────────────────┤
│ [✓ Erledigt]  [⊘ Ausgelassen]        │
│                                       │
│ ✏️ Notiz (optional)                   │
│ ┌──────────────────────────────────┐  │
│ │                                  │  │
│ └──────────────────────────────────┘  │
│                                       │
│ [   Speichern                     ]   │
│ [   Markierung & Notiz entfernen  ]   │
│                                       │
│ ───── Schnell-Aktion ─────            │
│ [⏭ Ab hier Rest des Tages offen   ]   │
│ [   lassen           (5 Items)    ]   │
│ Markiert dieses Item + alle danach    │
│ als ausgelassen. Einzeln rückgängig.  │
│                                       │
│ [   Abbrechen                     ]   │
└──────────────────────────────────────┘
```

Tap → alle Items ab hier werden mit „skipped" markiert + das aktuelle
Item bekommt eine Auto-Notiz „Rest des Tages offen — entspannte Pause"
(nur wenn der User nicht schon eine eigene Notiz hat).

### Konkret

- Reiseweg läuft, alle gut → Spontan-Pause:
  1. Tap auf „⋯"-Menü beim aktuellen Item
  2. „Ab hier Rest des Tages offen lassen"
  3. Sheet schließt sich, alle Items danach sind grau & durchgestrichen
  4. Stats-Badge im DayCard-Header zeigt automatisch das neue Verhältnis
- Wenn die Gruppe doch noch was schaffen will: einzelne Items per
  Circle-Button oder Action-Sheet wieder als erledigt markieren.

### Architektur

**`src/hooks/useItemState.ts`** — neue Methode:
```typescript
setRangeMark(dayIndex, fromIdx, toIdx, mark)
```
Atomare Bulk-Operation: setzt alle Items im Range mit derselben
Markierung in einem einzigen localStorage-Write. Spart 5–10
write-Operationen pro Tap und vermeidet React-Re-Render-Stürme.

**`src/components/trip/ItemActionSheet.tsx`** — neue Props:
- `onRestOfDayOpen?: () => void`
- `hasSubsequentItems?: boolean` — Button nur sichtbar wenn es
  weitere Items danach gibt (auf dem letzten Item wäre der Button
  sinnlos)
- `subsequentCount?: number` — fürs Label „(5 Items)"

**`src/components/trip/DayCard.tsx`** — Wiring:
- Reicht `onRestOfDayOpen(itemIndex)` an die Sheet weiter
- `hasSubsequentItems` automatisch berechnet aus
  `sheetIndex < day.items.length - 1`

**`src/components/tabs/ProgrammTab.tsx`** — Logik:
```typescript
onRestOfDayOpen={(fromIdx) => {
  itemState.setRangeMark(dayIdx, fromIdx, day.items.length - 1, "skipped");
  if (!itemState.get(dayIdx, fromIdx)?.note) {
    itemState.setNote(dayIdx, fromIdx, "Rest des Tages offen — entspannte Pause");
  }
}}
```

## Was funktioniert NICHT (in dieser Version)

- **Auto-Notiz ist deutsch fix**: „Rest des Tages offen — entspannte
  Pause". Nicht lokalisiert, weil die App momentan nur Deutsch ist.
- **Bulk-Action ist destruktiv** ohne Confirm-Dialog. Begründung: User
  kann jederzeit einzeln rückgängig machen (Circle-Button oder Sheet),
  und „Tag zurücksetzen"-Link macht alles auf einmal weg. Eine
  Confirm-Frage wäre unnötiger Reibung — Apple-Way „undo-friendly"
  statt „confirm-everything".
- **Phase 3 (Gruppen-Sync)** weiterhin offen — alle 5 Reisende sehen
  weiterhin eigene Sichten.
