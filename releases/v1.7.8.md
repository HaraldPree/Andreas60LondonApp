# v1.7.8 — Heutiger Tag default offen + Foto-Tag manuell zuordnen

**Datum**: 21.05.2026
**Status**: deployed
**Git Tag**: v1.7.8
**Bump-Grund**: PATCH — Zwei kleine UX-Verbesserungen aus Live-Feedback

## Was wurde geändert

### 1. Programm-Tab: heutiger Tag default offen (statt immer Tag 1)

User-Feedback: *„weiters ist immer tag 1 aufgeklappt es sollten
entweder der aktuelle tag ausgeklappt werden oder keiner"*

**Fix**: `defaultOpen={i === 0}` → `defaultOpen={day.isoDate === todayIso}`.

| Datum | Was passiert |
|---|---|
| Während der Reise | Heutiger Tag (matching isoDate) ist offen |
| Vor Reisestart / nach Ende | Kein Tag automatisch offen, User klickt selbst |

### 2. Fotos-Tab: Tag-Zuordnung manuell ändern (für „Unsortiert"-Fotos)

User-Feedback: *„bei den hochgeladenen fotos sollten es auch nach
tagen sein, entweder zum auswählen oder erkennen anhand wann das
bild aufgenommen wurde geht das"*

**Status**: Auto-Sort via EXIF gibt's schon (`assignToDay(exif.takenAt, days)`).
Fotos OHNE EXIF-Datum (Screenshots, Whatsapp-Fotos, Geräte ohne Datum)
landen in „Unsortiert". Bisher unverschiebbar.

**Fix**: PhotoDetail-Header zeigt jetzt das Tag-Label als **tappbaren
Button** mit dem Hinweis „(tippen)". Tap öffnet den `DayPickerModal`
(existiert schon, war bisher nur für die Karte) und ermöglicht
manuelle Tag-Zuweisung.

```
══════════════════════════════════════
📅 Tag 4 · Do, 21. Mai (tippen)  ← jetzt Button
14:32 · 21. Mai
══════════════════════════════════════
[Foto]
══════════════════════════════════════
```

Tap auf den Tag-Header → DayPickerModal mit allen 5 Tagen → Auswahl
→ Foto wandert in die DaySection (oder von „Unsortiert" raus).

### Wer profitiert
- **Andrea / Screenshot-Fotos**: EXIF-frei → landen in „Unsortiert" → jetzt manuell zuordnen
- **WhatsApp-weitergeleitete Fotos**: oft kein EXIF → jetzt zuordbar
- **Fotos mit falsch interpretierten Datum**: korrigierbar
