# v1.21.3 — Self-Destroying Service-Worker + RCMK-Stadt auf St. Valentin

**Datum**: 26.05.2026
**Status**: deployed
**Git Tag**: v1.21.3

## Anlass

Harald-Test nach v1.21.2-Deploy:

> v1.21.2 build 3ca3df6 ist aktiv, Wetter wird leider wieder nicht
> geladen. PWA deinstalliert und neu installiert, hat aber kein PW
> verlangt — daher muss er was aus dem Cache genommen haben.

**Diagnose**: klassischer „Service-Worker-Death-Loop". Der React-
Cleanup-Hook aus v1.21.2 ist im neuen Code, aber der **alte SW**
(v1.20.0/v1.21.0/v1.21.1) liefert weiterhin die **alte HTML-Page**
aus seinem Cache aus. Damit kommt der neue Cleanup-Code nie zur
Ausführung. Plus: PWA-Neuinstall ohne PIN-Prompt = der App-Cookie
und das Site-Caching sind noch im alten SW-Scope.

Plus User-Korrektur: RCMK-Filiale die er aktuell zum Testen nutzt
ist **St. Valentin** (nicht Linz, nicht Wien-Liesing).

## Was geändert

### 1) `public/sw.js` — Self-Destroying Kill-Switch (neu, ~85 Zeilen)

Echter Service-Worker (kein React-Hook!), direkt unter `/sw.js`
ausgeliefert. Browser prüft regelmäßig auf `/sw.js`-Updates (bei
jedem Page-Visit + alle ~24h periodisch). Sobald er diese Version
findet, wird sie installiert.

Bei `activate`:
1. **Alle Caches löschen** via `caches.keys()` + `caches.delete()` —
   inkl. Workbox-Precache, alle Runtime-Caches (`weather-api`,
   `tfl-api`, `app-api`, `static-images`, `osm-tiles`, `static-fonts`,
   `next-static`, `html-pages`)
2. **Sich selbst deregistern** via `self.registration.unregister()`
3. **Alle offenen Tabs reloaden** via `client.navigate(client.url)` —
   damit die neue, SW-freie HTML-Page geladen wird

Bewusst **kein `fetch`-Handler** — der SW lebt nur lang genug um
sich zu deregistern, interceptet während dieser Zeit nichts.

Wirkung: User mit altem SW kriegen beim nächsten Visit den
Kill-Switch geliefert → Cleanup läuft → Tab reloaded → ab da
SW-freier Browser → Wetter funktioniert direkt online.

### 2) `.gitignore` aufgeräumt

`public/sw.js` ist jetzt **bewusst** committed (manuelle Datei,

`fallback-*.js`, `*.map`) bleiben ignored.

Alte Vercel-Artefakte (`workbox-d50ca814.js`,

## Wie der Kill-Switch funktioniert

```
T=0   User öffnet App → alter v1.21.x-SW intercepts → alte Page geladen
T=1   Browser checkt /sw.js (parallel zur HTML-Anfrage)
T=2   Server liefert NEUE sw.js (Kill-Switch)
T=3   Browser sieht: SW-Code hat sich geändert → install Phase
T=4   install: skipWaiting() → wird sofort aktiv
T=5   activate: löscht alle Caches, unregistert sich
T=6   reload aller offenen Tabs → frische HTML-Page
T=7   Page lädt SW-frei → Wetter funktioniert online
```

Idempotent: bei späteren Visits ohne SW läuft nichts, kein Schaden.

## Was Harald tun soll

1. **Vercel-Deploy abwarten** (~2 Min)
2. **App öffnen** (egal ob PWA-Icon oder Browser-Tab)
3. Beim ersten Öffnen: Kill-Switch wird installiert + aktiviert
4. **Tab reloaded sich automatisch** (das ist Absicht — Kill-Switch
   triggert)
5. Nach dem Reload: SW ist weg, Wetter sollte direkt online laden
6. **Versions-Check im Info-Tab unten** → sollte „v1.21.3 · build [SHA]"
   zeigen
7. **Footer** → „Pilot-Reisebüro: … St. Valentin"

Falls **immer noch kein Wetter** nach automatischem Reload:
- 2. Mal manuell reloaden
- Falls dann immer noch tot: User-Browser hat hartnäckiges Site-Caching.
  Lösung: `Einstellungen → Webseitendaten löschen` für die Domain,
  dann App neu öffnen.
