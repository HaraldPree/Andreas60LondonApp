# v1.21.1 — Hotfix: Cross-Origin-APIs aus SW-Caching raus (Wetter geht wieder online)

**Datum**: 26.05.2026
**Status**: deployed
**Git Tag**: v1.21.1

## Anlass

Harald-Feedback nach v1.21.0-Deploy:

> Jetzt bin ich online und Wetter kann nicht geladen werden.

Genau das Risiko was beim Plan-Vorschlag erwähnt war: **Service-Worker-
Bugs sind heimtückisch**. v1.21.0 hatte Open-Meteo + TfL als
Cross-Origin-APIs im `runtimeCaching`-Pattern — Workbox-NetworkFirst
hat dabei offenbar Cross-Origin-Responses blockiert statt
durchzulassen.

## Ursache (Vermutung)

Workbox's NetworkFirst-Handler hat strikte Cross-Origin-Response-
Behandlung. Auch wenn Open-Meteo `Access-Control-Allow-Origin: *`
sendet, kann der SW die Response nicht sauber clonen/cachen wenn
spezifische Header fehlen oder die Browser-CORS-Policy strenger
interpretiert wird als gedacht. Resultat: Request schlägt fehl,
sowohl offline ALS AUCH ONLINE.

## Was geändert (Hotfix)

`next.config.js`:

**Entfernt** aus `runtimeCaching`:
- `^https://api\.open-meteo\.com/*` (Wetter)
- `^https://api\.tfl\.gov\.uk/*` (TfL Tube)

**Bleibt** (Same-Origin, unproblematisch):
- `/api/photos/list/...` — eigene API
- `/api/flight-status` — eigene API
- Alle static-Caching-Regeln aus v1.20.0

Die externen APIs fallen jetzt durch alle Cache-Regeln durch → SW
lässt sie unbehandelt durchs Netz. **Online funktioniert wieder
normal.**

## Was es kostet

- **Offline TfL-Status** ist auch wieder weg
- **Eigene App-Daten** (Foto-Liste, Flugstatus) bleiben offline-fähig

## Lessons Learned

- **Cross-Origin-Caching im SW ist riskanter als gedacht.** Auch bei
  CORS-freundlichen APIs (Open-Meteo) kann Workbox blockieren.
- **Lösung mittelfristig**: Wetter über `/api/weather`-Server-Proxy
  legen. Dann ist es Same-Origin und cache-fähig ohne CORS-Stress.
  Wird in einer späteren Iteration angegangen (eigene Mini-Iteration,
  vor oder nach v1.22.0).
- **Schrittweise war richtig**: Bug wurde nach 1 Tag entdeckt + in
  ~10 Min behoben. Bei Big-Bang-Implementierung wäre das mit 5
  weiteren Features mit-betroffen gewesen.
