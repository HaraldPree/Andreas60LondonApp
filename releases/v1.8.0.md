**Datum**: 22.05.2026 (Live, Abreise-Tag am Flughafen)

## Was ist neu

### 1. Gemeinsame Galerie nach Tagen gruppiert (Andrea-Feedback)

User-Feedback Andrea: *„zu unübersichtlich"* — die geteilten Fotos
landeten bisher als ein einziges Grid auf einem Stapel, schwierig zu
navigieren bei 30+ Bildern.

**Lösung**: Analog zur „Deine eigenen Fotos"-Sektion jetzt eine
**DaySection pro Reisetag**, ausklappbar:

```
═══════════════════════════════════════
👥 Gemeinsame Galerie                ⌃
─────────────────────────────────────
🟧 Tag 1 · Mo, 18. Mai          ⌃
   Anreise & West End
   8 Fotos

🟧 Tag 2 · Di, 19. Mai          ⌃
   Royal London + Cedric + Notting Hill
   14 Fotos

🟦 Tag 3 · Mi, 20. Mai          ⌃
   Leake Street + Borough + Tower Bridge
   12 Fotos

🟩 Tag 4 · Do, 21. Mai          ⌄ ← offen (heute)
   Borough Market + Uber Boat → Greenwich
   17 Fotos
   ┌──────────────────────┐
   │ [📷][📷][📷]            │
   │ [📷][📷][📷]            │
   └──────────────────────┘
═══════════════════════════════════════
```

Default-Logik: heutiger Tag (matching `isoDate`) ist offen, alle
anderen + „Unsortiert" sind zu. Konsistent mit dem FotosTab.

### 2. Goodbye-Reel mit Auto-Open am Abreise-Tag

User-Wunsch (am Flughafen): *„wir brauchen noch einen abschiedsgag —
ein kleines verabschiedungsgif — andrea wird nochmals gefeiert, dann
ggf die highlights der reise als kleines reel aus den fotos"*

**Geliefert**: Vollbild-Modal mit:
- Header: **„🎂 Andrea · 60 Jahre — Tschüss London"**
- Auto-Slideshow durch alle geteilten Fotos (chronologisch nach
  `takenAt`), 2.8 Sek pro Foto
- **Konfetti-Animation** im Hintergrund: 28 Emojis (🎂🎉🎊✨🎈👑💕🌟)
  die mit staggered Delay + random Duration von oben nach unten
  fallen + rotieren
- Pro Foto: Caption (falls vorhanden) + Uploader-Name + Aufnahme-Datum
- Controls: Pause / Play / Vor / Zurück
- Progress-Bar + Counter („7 / 23")
- Footer: „Frohe Heimreise — bis zur nächsten Reise mit Travel Live"

### Trigger-Verhalten

**Auto-Open beim ersten App-Open am Abreise-Tag** (matching Tag 5
isoDate = heute):
- 1.5 Sek nach App-Load öffnet das Reel automatisch
- localStorage-Flag `rcmk:reelSeen:london-2026` wird beim Close gesetzt
- Bei späteren App-Opens am selben Tag → kein Auto-Open mehr
- **Banner bleibt sichtbar** in Programm-Tab unter TripHero zum
  beliebigen Wiederansehen:

```
┌──────────────────────────────────────┐
│  🎬  LETZTER TAG — ABSCHIEDS-REEL    │
│  🎂 Andrea-Reel + Reise-Highlights   │
│  17 geteilte Fotos · Slideshow +     │
│  Konfetti                            │
└──────────────────────────────────────┘
```

Tap auf Banner → Reel öffnet sofort (kein localStorage-Check).

### Warum dieses Design

- **Auto-Open**: User-Wunsch „soll gleich aufgehen wenn die Info mit
  dem Update kommt" — also nach Reload erscheint es einmal von selbst
- **Banner bleibt**: User-Wunsch „den banner kannst du auch lassen zum
  öfter ansehen" — Re-Watch jederzeit möglich
- **localStorage**: pro Gerät persistiert, damit nicht jeder Tab-Wechsel
  das Reel nochmal aufpoppt
- **Vollbild + Gradient**: navy → gold Hintergrund passt zum Geburtstags-
  Anlass + Travel-Live-Branding

### Konfetti-Implementation (kein NPM-Package)

CSS-only via `<style jsx>` inline:

```css
@keyframes confetti-fall {
  from { transform: translateY(-10vh) rotate(0deg); opacity: 0; }
  10% { opacity: 1; }
  90% { opacity: 0.8; }
  to { transform: translateY(110vh) rotate(720deg); opacity: 0; }
}
```

28 absolut positionierte Emojis mit random `left`, `delay`, `duration`,
`size` (via `useMemo` einmal berechnet). Spart ~6 KB Bundle gegenüber
`canvas-confetti`.

### Filter-Logik

```ts
// Auto-Open:
if (isLastDay && sharedPhotos.length > 0 && !localStorage[reelSeenKey]) {
  setTimeout(() => setReelOpen(true), 1500);
}

// onClose:
setReelOpen(false);
localStorage.setItem(reelSeenKey, ISO-Datum);
```

## Was funktioniert NICHT (in dieser Version)

- **Eigene Fotos nicht im Reel** — bewusste Entscheidung. Shared Photos
  sind die kuratierten Gruppen-Erinnerungen, alle 5 sehen sie. Eigene
  Fotos brauchen `useBlobUrl` pro Item (Hook-in-Loop-Problem) und sind
  oft sehr persönlich (Selfies etc.). Falls gewünscht: v1.8.1 mit
  Toggle „auch eigene Fotos zeigen".
- **Kein Musik-Hintergrund** — Web-Audio würde Auto-Play-Restrictions
  ausgelöst (iOS Safari blockt). Reels bleiben silent. Apple-Way OK.
- **Kein Export als Video/GIF** — der Reel läuft nur in der App, ist
  keine teilbare Datei. Falls gewünscht: User kann am Handy via
  Bildschirmaufnahme aufzeichnen.
- **Reel kommt nur am letzten Reisetag** (matching `isoDate`). Vor
  oder nach der Reise: kein Banner, kein Auto-Open.
