# v1.3.1 — Apple-Way Polish-Bundle

**Datum**: 19.05.2026
**Status**: deployed
**Git Tag**: v1.3.1
**Bump-Grund**: PATCH — UX-Politur aus dem UX-Audit, keine neuen Features

## Was ist neu

Sechs gezielte Apple-HIG-konforme Verbesserungen aus
`todo/v1.1.0-ux-audit.md`. Alles in einem Bundle deployed, jeder Punkt
für sich klein, zusammen spürbar smoother.

### 1. KI-Companion-Sparkle ruhig gestellt

**Datei**: `src/components/companion/CompanionWidget.tsx`

- `animate-pulse` weg
- grüner Notification-Punkt weg

Wirkte vorher wie Werbe-Notification (Apple-Antipattern). Jetzt:
statischer Sparkle-Button mit Gold-Gradient.

### 2. Touch-Targets auf Apple-HIG-Minimum

**Dateien**:
- `src/components/layout/Header.tsx` — Back-Link von ~28px → 44pt
  Höhe (sichtbarer Padding-Korrektur via negativer Margin)
- `src/components/layout/ScrollToTop.tsx` — 40px → 44px (w-11 h-11)

Apple HIG fordert 44pt × 44pt für alle tappbaren Elemente. Beide
waren knapp drunter.

### 3. iOS-Upload-Hilfe-Hinweis collapsible

**Datei**: `src/components/photos/PhotoUpload.tsx`

Der iPhone-Tipp (Galerie nicht da → Safari-Berechtigung prüfen) war
in v1.2.3 als statischer Block eingeführt — auf kleinen iPhones SE
hat das das Menü zu lang gemacht. Jetzt: collapsible „Galerie
erscheint nicht? Hilfe für iPhone" mit `HelpCircle`-Icon — nur
sichtbar wenn der User aktiv drauf tippt.

### 4. Foto-Galerie zeigt „Stand: HH:MM"

**Datei**: `src/components/photos/SharedGallery.tsx`

`lastUpdatedAt` aus dem `useSharedPhotos`-Hook (eingeführt in v1.2.3,
bisher nicht angezeigt) erscheint jetzt als kleine Mono-Schrift im
Footer der Galerie: „Stand: 14:32". Hilft beim Verstehen ob die
gerade gezeigten Fotos der aktuelle Stand sind oder Cache.

### 5. Firefox-Mikro-Tooltip-Fallback

**Datei**: `src/components/companion/ChatPanel.tsx`

Firefox Android & Desktop haben keine Web Speech API. Vorher war
in dem Fall der Mikrofon-Button komplett versteckt — kein Hinweis
warum. Jetzt: ein gedimmter Mikro-Button mit `?`-Charakter, Tap
zeigt einen Tooltip „Spracheingabe in diesem Browser nicht verfügbar.
Tipp: 🎤 auf deiner Handy-Tastatur drücken — das funktioniert
überall."

Variante A aus `todo/firefox-mic-fallback.md`. Variante C
(MediaRecorder + Whisper) bleibt für später falls echtes Bedarf.

Bonus: der existierende Mikro-Button hat jetzt auch 44pt (w-9 → w-11).

### 6. Reservation-Status: Action-Sheet statt Cycle-Tap

**Dateien**:
- `src/hooks/useReservations.ts` — `setStatus(id, next)` hinzugefügt
- `src/components/reservations/ReservationCard.tsx` — Status-Balken
  öffnet jetzt Sheet, „(tippen zum Ändern)"-Hint entfernt
- `src/components/reservations/ReservationTracker.tsx` — Sheet wiring
- `src/components/reservations/ReservationStatusSheet.tsx` *(neu)* —
  Apple-Action-Sheet mit drei Optionen (offen / reserviert / erledigt),
  jeweils mit Icon + Beschreibung, aktive Option markiert

Vorher: Tap zykliert offen→reserviert→erledigt→offen. War unintuitiv
(„wo ist erledigt direkt?") und brauchte einen Hint im UI. Jetzt:
Tap öffnet Sheet, User sieht alle drei Optionen mit Erklärung,
wählt direkt. Apple-Way ✓.

Action-Sheet nutzt `useDismissOnBack` aus v1.2.3 → Swipe-Back
schließt sauber.

## Was funktioniert NICHT (in dieser Version)

- **Action-Sheet zeigt nur 3 Optionen** — keine zusätzliche „Notiz pro
  Reservation"-Funktion (das wäre eigenes Feature, nicht Polish).
- **Firefox-Mikro-Tooltip schließt nicht automatisch** beim nächsten
  Tap außerhalb. User muss erneut auf den Button tippen um zu
  schließen. Wäre eine kleine Verbesserung (Click-Outside-Handler)
  für v1.3.2 falls nervig.
- **PhotoUpload-Hilfe-Hinweis** ist still für alle, nicht erst nach
  erkanntem iOS. Etwas verschwendete Realestate aber akzeptabel weil
  collapsed by default.
