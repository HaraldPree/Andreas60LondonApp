## Was ist neu

### 1. Type-Erweiterung (`src/types/photo.ts`)

```ts
export type MediaType = "photo" | "video";

interface PhotoEntry / PhotoMeta {
  ...
  mediaType?: MediaType;         // default "photo"
  videoDurationSec?: number;     // bei Videos für Display-Pill
}
```

Backwards-kompatibel: bestehende Foto-Einträge ohne `mediaType` werden
als `"photo"` interpretiert.

### 2. Video-Processing (`src/lib/photoProcessing.ts`)

Neue Helper:
- `isVideoFile(file)` — MIME + Extension-Check (mp4 / mov / webm / m4v)
- `videoPreflight(file)` — Size-Check (max 100 MB), Format-Check
- `processVideoForStorage(file)` — extrahiert Cover-Frame, baut Thumbnail
- `videoTakenAt(file)` — fallback auf `file.lastModified` (Videos haben kein EXIF wie Fotos)

**Cover-Frame-Extraktion**:
1. `<video>`-Element wird kurz im DOM mounted (opacity 0, off-screen)
2. Metadaten laden → Dimensionen + Dauer bekannt
3. Seek zu `min(1s, 10% der Dauer)` — vermeidet typische schwarze Anfänge
4. Frame auf Canvas zeichnen, als JPEG-Blob komprimieren (max 1500 px)
5. Thumbnail aus Poster (max 320 px)

Original-Video bleibt **unkomprimiert** im `fullBlob` — Browser haben
keine sinnvolle Video-Komprimierungs-API. Daher 100 MB Hard-Limit pro
Datei (IndexedDB-Quota + Mobile-RAM-Schutz).

### 3. Upload-Pipeline (`src/hooks/usePhotos.ts`)

Branch im `upload`-Handler:
```ts
if (isVideoFile(file)) {
  // Video-Pfad: videoPreflight + processVideoForStorage
  // PhotoEntry mit mediaType: "video", videoDurationSec
} else {
  // Photo-Pfad (bestehend, unverändert)
}
```

Fehler werden wie bisher in `uploadErrors` gesammelt und angezeigt.

### 4. UI-Erweiterungen

**`PhotoUpload`**: `accept="image/*,video/*"` statt nur `image/*` —
Galerie-Picker zeigt jetzt auch Videos.

**`PhotoCard`**: bei `mediaType === "video"`:
- Schwarzer 25 %-Overlay
- Weißer Play-Button im Zentrum (Lucide `Play`-Icon)
- Dauer-Pill unten links (mm:ss oder h:mm:ss)

**`PhotoDetail`**: bei `mediaType === "video"`:
- `<video src={fullUrl} controls playsInline>` statt `<img>`
- Schwarzer Hintergrund für saubere Letterbox

### 5. Adapter-Update (`src/lib/exportPhotosAdapter.ts`)

`combineForExport` filtert **Videos heraus** bevor sie an PDF/ZIP-Generator
gehen. Sonst würde der Generator versuchen ein Video als JPEG zu rendern
und crashen. Videos sind also nur in der Galerie sichtbar, nicht im
Export — das kommt in v1.12.2.

## Was funktioniert

- ✅ Video hochladen (mp4 / mov / webm / m4v, max 100 MB)
- ✅ Video in Galerie mit Play-Icon + Dauer-Pill

## Bekannte Limits

- **iOS Safari**: kann manche HEVC-Videos nicht in Canvas zeichnen → Cover-Frame
  schlägt fehl, Video wird nicht akzeptiert. Workaround: Video als MP4 H.264
  exportieren (iOS-Foto-App: „Beim Übertragen automatisch konvertieren")
- **IndexedDB-Quota**: bei vielen Videos (>5 GB total) kann Browser
  Daten löschen. Hard-Limit pro Video bei 100 MB schützt davor
- **Video-Preview im Selection-Sheet** (v1.11.2): zeigt Poster-Frame
  korrekt, aber kein „Video"-Badge — kommt in v1.12.2 zusammen mit
  Export-Integration

## Migrations-Hinweise
