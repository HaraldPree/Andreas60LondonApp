## Bugs gefixt

### Bug A — DaySeparator wraps zu 2 PDF-Seiten

`<Page>` in react-pdf wrappt Content automatisch wenn Overflow.
Bei langem `day.summary` + `justifyContent: center`-Layout sprengt's
die Höhe → Page-2 erscheint, oft mit fragmentiertem Text.

**Fix in `src/lib/photoBookPdf.tsx`**:

1. `wrap={false}` auf **alle 6 Page-Komponenten** (Cover, Day-Separator,
   Unsorted-Separator, PhotoPair, SinglePhoto, Closing)
2. Neue `truncate(s, 320)` Helper-Funktion
3. `day.summary` wird vor Render auf 320 Zeichen gekürzt (mit „…" Suffix)

### Bug B — Phantompages durch nested Array-Returns

Im Document Main:

```jsx
{daysWithPhotos.map((dayIdx) => {
  ...
  return [
    <DaySeparatorPage .../>,
    ...pages,
  ];  // ← Array<Array<Page>>
})}
```

React-PDF rendert verschachtelte Arrays anders als ein flaches Array — pro
Day-Block kamen 1-2 leere Phantompages dazu.

**Fix**: Flat array vorbauen, einmal returnen:

```jsx
{(() => {
  const allPages = [];
  for (const dayIdx of daysWithPhotos) {
    allPages.push(<DaySeparatorPage .../>);
    ...
  }
  if (unsorted.length > 0) {
    allPages.push(<UnsortedSeparatorPage />);
    ...
  }
  return allPages;
})()}
```

### Bug C — „Direct-Download" Anchor verursacht about:blank

In v1.10.1 hatte ich neben dem blauen Share-Button noch einen kleinen
Fallback-Anchor `<a href="blob:" download>` als „Alternativ: Direkt-Download
(Desktop / Chrome)" gelassen. Harald hat den vermutlich versehentlich
geklickt → Samsung Internet zeigt `about:blank` (klassischer Browser-Bug
mit Blob-URLs).

**Fix in `PdfBookExportButton.tsx` + `PhotoBookExportButton.tsx`**:

- Direct-Download-Anchor **komplett entfernt**
- Nur noch EIN Speichern-Button (`handleSave()` aus v1.10.1)
- Hint-Text vereinfacht: „Auf Handy öffnet sich Teilen-Sheet → ‚In Dateien
  speichern' oder direkt an WhatsApp/Mail"
- `handleSave` versucht intern beides: erst Share-API, dann anchor-Click
  als Last-Resort — User sieht das aber als **einen** Button

## Was funktioniert NICHT (bekannte Limits)

- **Wenn ein Day-Summary genau 320+ Zeichen ist**: wird hart gekürzt mit „…".
  Lange Geschichten gehören dann eher in `tips` oder `weatherHint`,
  nicht in `summary`
- **Wenn navigator.share auf Desktop fehlt**: handleSave fällt auf
  programmatischen Anchor-Click zurück — funktioniert auf allen Desktop-
  Browsern. Auf Samsung Internet ohne Share-API: kein Workaround mehr,
  aber das ist sehr selten
