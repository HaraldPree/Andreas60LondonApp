# v1.7.6 — Fotos-Tab: Tage + Gemeinsame Galerie ausklappbar

**Datum**: 20.05.2026
**Status**: deployed
**Git Tag**: v1.7.6
**Bump-Grund**: PATCH — UX-Fix gegen Endlos-Scroll (Lukas-Feedback)

## Was wurde geändert

User-Feedback (Lukas, Live während London-Reise):
> „bei der Fotoansicht sollten die verschieden ansichten je Tag und
> die geteilten Bilder zum ausklappen sein — es sind schon so viele
> Bilder da, da muss so viel gescrollt werden"

Stimmt. 5 Reisende × 5 Tage × ~10 Fotos pro Person + Gemeinsame Galerie
= mehrere Hundert Thumbnails offen auf einmal. Nicht handlich.

### Lösung: alles collapsible

#### 1. Gemeinsame Galerie ausklappbar

```
┌─────────────────────────────────────────┐
│ 👥 Gemeinsame Galerie              ⌃    │
│    Was andere Mitreisende freigegeben…  │
└─────────────────────────────────────────┘
  (klicken → schließt sich)
```

Klick auf den Header → kollabiert. Default beim ersten Öffnen: offen
(weil's eine prominente Sektion ist, soll gleich sichtbar sein).

#### 2. Pro Tag eigene DaySection ausklappbar

```
┌─────────────────────────────────────────┐
│ 🟨 (Tag-Farbleiste)                      │
├─────────────────────────────────────────┤
│ TAG 1 · MO, 18. MAI                     │
│ Anreise & West End                       │
│ 7 Fotos                             ⌃   │
└─────────────────────────────────────────┘
  (zugeklappt — kein Thumbnail-Grid sichtbar)
```

Default-Logik: **nur der heutige Tag ist offen**, alle anderen
zugeklappt. „Unsortiert"-Sektion ebenfalls zu.

Beispiel mit heute = 20.05. (Mi):

| Tag | isoDate | Default |
|---|---|---|
| Tag 1 (Mo 18.5.) | 2026-05-18 | zu |
| Tag 2 (Di 19.5.) | 2026-05-19 | zu |
| **Tag 3 (Mi 20.5.)** | **2026-05-20** | **auf** ← heute |
| Tag 4 (Do 21.5.) | 2026-05-21 | zu |
| Tag 5 (Fr 22.5.) | 2026-05-22 | zu |
| Unsortiert | — | zu |

Wenn ihr außerhalb der Reise drauf seid (vorher / nachher): alles zu,
ihr klickt rein.

### Apple-Way

Klar definierte „Falten" wie iOS Photos / iOS Settings — Header bleibt
sichtbar mit Title + Counter („7 Fotos"), Inhalt nur auf Anforderung.

## Was funktioniert NICHT

- Collapse-State wird NICHT persistiert. Beim erneuten Öffnen des Tabs
  oder App-Reload sind die Standards wieder aktiv (heute auf, andere zu,
  Galerie auf). Für jetzt OK — Persistenz-State würde localStorage
  verbrauchen + mehr Komplexität.
- Auf-/Zu-Animation ist abrupt (kein Framer-Motion-AnimatePresence).
  Apple-Way wäre weicher. Falls als störend empfunden, in v1.7.7
  nachschärfen.
