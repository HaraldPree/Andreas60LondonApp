# v1.16.1 — UX-Polish nach Erst-Test (Anlass-Zeile, Event-Banner, Reise-Fragen sichtbar)

**Datum**: 26.05.2026
**Status**: deployed
**Git Tag**: v1.16.1

## Anlass

Harald-Feedback nach v1.16.0-Live-Test, drei konkrete Verbesserungen:

1. **„NPS ist super, ich wurde jetzt aber nicht zur Reise abgefragt"** —
   die zwei Sekundär-Fragen („Inhalt der Reise" / „Organisation der
   Reise") waren ausklappbar default ZU. User hat sie übersehen.
2. **TripHero**: „60. Geburtstag von Andrea" stand inline neben
   „London" — gehört in eigene Zeile.
3. **EventBanner** (Chelsea Flower Show etc.): ist nach Reise-Ende
   irrelevant, soll ausgeblendet sein.

## Was geändert

### 1) Feedback-Karte: Sekundär-Fragen Default OFFEN

`src/components/feedback/FeedbackCard.tsx`:

- `useState(true)` für `secondaryOpen` statt vorher abhängig von
  bestehenden Werten
- Toggle-Text geändert: „Auch die Reise bewerten (optional, 2 kurze
  Fragen)" — explizit auf die ZWEI Fragen hingewiesen

User sieht jetzt sofort:
- Hauptscore (App-NPS, Pflicht)
- Beide Reise-Achsen aufgeklappt (Inhalt + Organisation)

### 2) Header: Anlass in eigener Zeile

`src/components/layout/Header.tsx`:

```tsx
// vorher
<h1>London<span className="ml-2 text-base">{occasion}</span></h1>

// neu
<h1>London</h1>
{occasion && <p className="font-display text-base mt-0.5">{occasion}</p>}
```

Hierarchie: Stadt = H1, Anlass = Subtitle direkt darunter, dann erst
Datum. Anlass kriegt eigene visuelle Würde.

### 3) EventBanner nach Reise-Ende ausblenden

`src/components/tabs/ProgrammTab.tsx`:

```tsx
{!isPastTrip && <EventBanner trip={trip} />}
```

Reise vorbei → Chelsea Flower Show 19.–23.5. interessiert keinen mehr.
Banner verschwindet automatisch ab dem Tag nach Reise-Ende. Während
und vor der Reise weiterhin sichtbar.
