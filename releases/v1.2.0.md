# v1.2.0 — Programm-Varianten-Switch (Wetter-Anpassung)

**Datum**: 21.05.2026
**Status**: deployed
**Git Tag**: v1.2.0
**Bump-Grund**: MINOR — neues User-Feature

## Was ist neu

Der KI-Companion hat aufgrund der Wetter-Prognose (Dienstag Regen, Donnerstag
sonnig) eine **alternative Programm-Variante** vorgeschlagen. Das ursprüngliche
Programm bleibt erhalten — jede:r Reisende kann pro Gerät zwischen den
beiden Versionen wechseln.

### Konkret geänderte Tage in der London-2026-Reise

**Dienstag 19. Mai** (Index 1) — Regen-Anpassung:
- Verkürzter Royal Walk am Vormittag (Big Ben/Westminster gestrichen)
- 13:00 Cedric Grolet bleibt fix (Reservation unverändert)
- Nachmittag: V&A Museum statt Notting Hill (indoor, gratis)
- Abend: Pub-Dinner in Soho (kurze Wege bei Regen)

**Donnerstag 21. Mai** (Index 3) — Sonnentag-Bonus:
- Vormittag: Thames Clipper nach Greenwich (umgeht Tube-Streik elegant)
- Greenwich-Highlights wie geplant (Cutty Sark, Royal Observatory, Painted Hall)
- DLR zurück (fährt trotz Tube-Streik)
- **Spätnachmittag: Bus 23 nach Notting Hill / Portobello** —
  vom Dienstag verschobenes Highlight im warmen Abendlicht
- Pub-Dinner Notting Hill

**Montag / Mittwoch / Freitag**: unverändert (Anreise / City-Tag / Abreise).

### Architektur

Neuer Trip-Type:
```ts
{
  days: Day[],                          // wie bisher
  alternativeDays?: Array<Day | null>,  // gleiche Länge, null = fallback
  alternativeDaysMeta?: {
    label: string,
    description: string,
    source: string,
    affectedDayIndices: number[]
  },
  defaultVariant?: "original" | "alternative"
}
```

Neuer Hook `useTripVariant(tripSlug, defaultFromTrip)` — speichert die Wahl
in localStorage pro Reise. Fallback wenn leer: `trip.defaultVariant ?? "original"`.

Neue Komponente `TripVariantSwitcher`:
- Erscheint nur wenn der Trip alternative Tage hat
- 2 Pills: „Original" / Label der Alternative
- Aufklappbarer Detail-Bereich: was wurde geändert, von wem vorgeschlagen,
  welche Tage betroffen

`TripPageClient` berechnet `effectiveTrip` und gibt es an ALLE Tabs +
Companion-Widget weiter. So sehen Karte, Reservierungen, KI-Chat
automatisch das aktive Programm — keine Inkonsistenzen.

`ProgrammTab` rendert den Switcher direkt unter dem TripHero — vor
Wetter-Widget — damit User sofort sehen welche Version sie lesen.

### London-2026 Default

`defaultVariant: "alternative"` — d.h. **neue User sehen by default
das Wetter-angepasste Programm**, weil das aktuell zutreffend ist.
Wer's umschalten will: ein Tap auf „Original" Pill.

### Was die Mitreisenden gleich sehen

```

├─────────────────────────────────────┤

```

## Was funktioniert NICHT (in dieser Version)

- **Pro Gerät, nicht synchron**: jede:r entscheidet selbst. Wenn Harald
  „Original" wählt, sehen die anderen 4 trotzdem ihre eigene Wahl.
  Konsistent mit Lukas' früherem Wunsch (alles für alle gleich) wäre ein
  Backend-Switch — kommt in einem späteren Release falls gewünscht.
- **Mehrere parallele Varianten**: aktuell nur EINE Alternative pro Trip.
  Architektur könnte später auf mehrere erweitert werden (z.B. „Regen",
  „Streik", „Spontan"-Varianten je nach Situation).
- **Reservation-Tab zeigt für beide Varianten gleich**: das ist OK weil
  die Cedric-Reservation in beiden Versionen identisch bleibt.
- **Karte zeigt mapPoints des aktiven Tages**: das passt, aber Notting
  Hill / V&A wandern je nach Variante zwischen Tagen. Filter „Tag 2" =
  Original-Tag-2 vs. Alternative-Tag-2 unterscheidet sich.

## Migrations-Hinweise

Keine. Frontend-only, kein Backend-Setup nötig. Beim ersten Öffnen sieht
jede:r die Alternative (Default für diese Reise) — kann jederzeit
umschalten.
