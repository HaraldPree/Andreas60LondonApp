# v1.21.0 — Service-Worker Schritt b (API-Caching: Wetter, TfL, Foto-Liste, Flight)

**Datum**: 26.05.2026
**Status**: deployed
**Git Tag**: v1.21.0

## Anlass

Harald-Feedback nach v1.20.0-Test:

> Webseite lädt — Wetter konnte nicht geladen werden, sonst ist
> überall was da. Auch kein gecachtes Wetter?? bei Erleben

Genau das was Schritt a nicht abgedeckt hat: **API-Calls**. Die statischen
Sachen (Layout, Bilder, Karten, Tabs) waren gecacht — aber das Wetter
geht jedes Mal frisch an Open-Meteo, und das ist offline tot.

v1.21.0 = Schritt b der drei-stufigen Service-Worker-Etappe.

## Was ist neu in `next.config.js`

**3 neue Caching-Regeln** vor die statischen Patterns einsortiert
(erste passende Regel gewinnt):

| API | URL-Pattern | Strategie | Cache | Begründung |
|---|---|---|---|---|
| **Open-Meteo Wetter** | `api.open-meteo.com/*` | NetworkFirst (3s Timeout) | 30 Min, max 50 | Wetter ändert sich nicht alle 5 Min, aber online wenn möglich |
| **TfL Tube-Status** | `api.tfl.gov.uk/*` | NetworkFirst (3s Timeout) | 10 Min, max 30 | Streiks/Delays zeitkritisch — offline lieber letzter Stand als nichts |
| **Eigene App-API** (Read-Only) | `/api/photos/list/...`, `/api/flight-status` | StaleWhileRevalidate | 1 Std, max 50 | User sieht sofort letzten Stand, Hintergrund-Update für nächstes Mal |

### Bewusst NICHT gecacht — bleibt online-frisch

| API | Grund |
|---|---|
| `/api/version` | Update-Detection muss live sein, sonst sieht `useVersionCheck` keine neuen Versionen |
| `/api/chat` | LLM-Call (Anthropic API-Kosten), jede Frage frisch |
| `/api/photo-narrate` | LLM-Vision-Call (Kosten) |
| `/api/research/events` | LLM-Recherche-Call (Kosten) |
| `/api/photos/share` | POST — Workbox cached POST eh nicht standardmäßig |
| `/api/login`, `/api/logout` | Auth-Pfad — niemals cachen |

## Verhalten nach diesem Release

| Szenario | Vorher (v1.20.0) | Nach v1.21.0 |
|---|---|---|
| Online, Wetter laden | Frischer Request | Frischer Request + cached für später |
| Offline, App schließen + neu öffnen | „Wetter konnte nicht geladen werden" | Wetter aus Cache (max 30 Min alt) sichtbar |
| Offline, Tube-Tab öffnen | Fehler | Cached Status (max 10 Min) sichtbar |
| Offline, geteilte Galerie | Leerer Spinner | Letzte bekannte Galerie sichtbar |
| Offline, Flight-Status | Fehler | Letzter bekannter Status sichtbar |
| Online, Update-Banner | Funktioniert | Funktioniert (version nicht gecacht) |
| AI-Companion-Chat | Funktioniert online | Bleibt online-pflichtig (LLM-Cost) |

## Wie testen (neuer Schritt-b-Test)

1. **Online**: App öffnen, Wetter ist sichtbar — einmal warten bis es lädt
2. **Geteilte Galerie** öffnen (Erinnern-Tab) — Fotos werden geladen
3. **Flugstatus** öffnen (Info-Tab) — Status wird angezeigt
4. **App schließen**, **Flugmodus an**, **App wieder öffnen**
5. Erleben-Tab → **Wetter sollte sichtbar sein** mit letztem Online-Stand
6. Erinnern-Tab → **Galerie zeigt letzten Stand** der geteilten Fotos
7. Info-Tab → **Flug-Status mit letztem bekannten Stand**

Wenn das funktioniert: alles was häufig genutzte Daten sind, hat
offline-Fähigkeit. Bleibt v1.22.0 für die **Foto-Thumbnails** der
geteilten Galerie (Vercel-Blob-URLs).

## Was bleibt für v1.22.0

Noch nicht in v1.21.0 abgedeckt:
- **Geteilte Foto-Thumbnails** (Vercel-Blob-URLs): Galerie zeigt
  offline „⚠ Bild kaputt" weil die Thumbnail-URLs nicht gecacht sind
- **Geteilte Foto-Vollauflösung** (auch Vercel-Blob)

Diese URLs sehen aus wie `https://[hash].public.blob.vercel-storage.com/...`
und brauchen eine eigene Cache-Regel. Kommt im finalen Schritt c.
