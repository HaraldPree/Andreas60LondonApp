# v1.6.0 — Leger vs. Original-Programm umschalten

**Datum**: 20.05.2026
**Status**: deployed
**Git Tag**: v1.6.0
**Bump-Grund**: MINOR — Programm-Struktur umgestellt, Variant-Switcher reaktiviert mit neuer Semantik

## Was ist neu

Auf User-Wunsch: das **Programm ist wieder einheitlich** für alle 5
Reisenden (kein per-Gerät-Editor mehr), dafür gibt es einen Umschalt-
Switch zwischen zwei kuratierten Versionen:

```
┌──────────────────────────────────────────┐
│ Aktive Programm-Version                  │
│ Leger                                    │
│ Entspannte Live-Version — mehr Pause,    │
│ weniger Stress                           │
├──────────────────────────────────────────┤
│  [✓ Leger]    [ Original-Programm ]      │
├──────────────────────────────────────────┤
│ ℹ️ Was wurde geändert?                   │
└──────────────────────────────────────────┘
```

### „Leger" (Default — was die Reisegruppe tatsächlich macht/gemacht hat)

- **Di 19.05.** — Royal Walk + Cedric + Notting Hill + Italienisch
- **Mi 20.05.** — Leake Street Tunnel + Borough Market + Tower Bridge + Garden at 120 (Skyline-Sundown)
- **Do 21.05.** — Greenwich per Thames Clipper + Royal Observatory + Greenwich Market

Geh-Strecke pro Tag deutlich reduziert, ein Markt + Highlights + Aussicht pro Tag, viel Pause-Möglichkeit. Tag 2 hard-coded wie tatsächlich gelaufen.

### „Original-Programm" (zum Umschalten)

- **Di** — Royal Walk + Cedric + Hyde Park + Harrods + V&A + Notting Hill + Pub-Dinner (15 Items, ursprüngliche Planung)
- **Mi** — City Day: Barbican + Guildhall + St. Paul's + Tower of London + Tower Bridge + HMS Belfast + Millennium Bridge + Halcyon Gallery + Ronnie Scott's
- **Do** — Greenwich Vollprogramm: Cutty Sark + Greenwich Market + Royal Observatory + Painted Hall + Canary Wharf + Crossrail Roof Garden

Geh-intensiver, mehr Highlights, mehr Eindrücke pro Tag.

### Wo umschalten?

Im Programm-Tab oben unter dem Trip-Hero. Zwei Pills, Tap = umschalten. Auswahl pro Gerät persistiert in localStorage.

## Was wurde technisch geändert

### Trip-Daten (`src/data/trips/london-2026.ts`)

| Property | Inhalt v1.5.1 | Inhalt v1.6.0 |
|---|---|---|
| `days[1]` (Di) | „Vormittag erledigt, Rest offen" Live-Version | Final-Version Di („Royal London + Cedric + Notting Hill") |
| `days[2]` (Mi) | City Day Vollprogramm | **Neu**: Leger Mi (Leake Street + Borough Market + Tower Bridge + Garden at 120) |
| `days[3]` (Do) | Greenwich + Canary Wharf | **Neu**: Leger Do (Thames Clipper + Cutty Sark + Greenwich Market + Royal Observatory) |
| `alternativeDays[1]` | null | **Neu**: Original Di mit Hyde Park / Harrods / V&A / Pub |
| `alternativeDays[2]` | null | **Neu**: Original Mi mit Tower of London / St. Paul's etc. |
| `alternativeDays[3]` | „Sonnentag-Bündelung" | **Neu**: Original Do mit Painted Hall + Canary Wharf |
| `alternativeDaysMeta` | Wetter-spezifisch | Neutral: `label`, `originalLabel`, `*Subtitle` |
| `defaultVariant` | "original" | unverändert: "original" (= rendert `days` = Leger) |

### TypeScript-Type (`src/types/trip.ts`)

`alternativeDaysMeta` erweitert um:
- `originalLabel?: string` — Pill-Label für die default-Variante
- `originalSubtitle?: string` — Sub-Text wenn Original-Variante aktiv
- `alternativeSubtitle?: string` — Sub-Text wenn Alternative aktiv

Damit ist der Switcher generisch nutzbar (vorher Wetter-spezifisch
fest „Original" + „Regen-Variante" verdrahtet).

### TripVariantSwitcher (`src/components/trip/TripVariantSwitcher.tsx`)

Komplettes Refactor:
- Icons jetzt generisch (`Leaf` + `Map`) statt wetter-spezifisch
  (vorher CloudRain/Sun)
- Beide Pill-Labels über `meta` konfigurierbar
- Subtitles in der Header-Box je nach aktivem Modus
- Touch-Targets auf 44pt (Apple HIG, schon aus v1.3.1)

### In-App-Editor deaktiviert (`src/components/tabs/ProgrammTab.tsx`)

- `useItemState`-Hook nicht mehr aufgerufen
- Keine Callbacks (`itemStateFor`, `onToggleItemDone`, `onCommitItemState`,
  `onClearItem`, `onClearDay`, `onRestOfDayOpen`) mehr an `DayCard`
  durchgereicht
- Resultat: Circle-Button / Menu-Button / Stats-Badge / „Tag zurücksetzen" /
  „Ab hier offen lassen" — alles verschwindet
- **Code bleibt im Repo**: `useItemState.ts`, `ItemActionSheet.tsx`,
  Day-Card-Props sind weiterhin da. Reaktivierung später (vermutlich mit
  Gruppen-Sync = Phase 3) durch erneutes Wiring in `ProgrammTab`.

## Was funktioniert NICHT (in dieser Version)

- **Variant-Wahl pro Gerät**, nicht gruppen-synchronisiert. Wenn Harald
  „Original" wählt, Andrea aber „Leger" lässt, sehen die zwei
  unterschiedliche Programme. Hinweis dazu wird in den Switcher-Details
  angezeigt. Konsistenz mit dem früheren Pattern aus v1.2.0.
- **In-App-Editor-localStorage-Daten** bleiben in localStorage stehen
  (rcmk:items:london-2026) — werden aber ignoriert weil UI deaktiviert.
  Bei Wiederaktivierung würden alte Markierungen wieder erscheinen.
  Kein Cleanup-Job — kann der User manuell in DevTools löschen wenn
  gewünscht.
- **Rainy-Alternative-Block** an Tag 3 + Tag 4 weggefallen (war mit
  altem City Day / Greenwich-Vollprogramm verknüpft). Bei Bedarf später
  pro Leger-Tag neu einbauen.

## Migrations-Hinweise
