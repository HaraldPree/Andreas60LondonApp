# v1.1.0 — Foto-Sharing Frontend (Phase 1 komplett)

**Datum**: 21.05.2026
**Status**: deployed (Code) — aktiviert sobald Vercel Blob + KV im Dashboard verbunden
**Git Tag**: v1.1.0
**Bump-Grund**: MINOR — neues User-Feature, abwärtskompatibel

Zweites von zwei Releases zur **Phase-1 Foto-Sharing**. v1.0.1 lieferte
das Backend, v1.1.0 vervollständigt die User-Erfahrung.

## Was ist neu

### Neue UI-Komponenten

**`SharingConsentModal`** (`src/components/photos/SharingConsentModal.tsx`)
- Einmaliges Modal vor erstem Foto-Sharing
- Zwei Pflicht-Checkboxen: AGB v1.0 + Datenschutz v1.0
- Beide müssen aktiv angeklickt sein — nichts ist vor-gewählt
- Bei Annahme: `acceptCurrentVersions()` schreibt Marker in localStorage
- Bei Abbruch: Foto bleibt 🔒 Privat
- Erscheint nicht mehr nachdem Einwilligung erteilt ist (außer AGB-Version
  wird zukünftig erhöht — dann erneute Annahme nötig)

**`VisibilitySelector`** (`src/components/photos/VisibilitySelector.tsx`)
- 3-Radio-Komponente: 🔒 Privat / 🎂 Geburtstagskind / 🌐 Gruppe
- Default-Wahl wird vom Aufrufer gesetzt (immer `private`)
- `compact`-Variante für In-Galerie-Toggle
- Optionaler `celebrantName` — wenn gesetzt, wird der Label personalisiert
  („🎂 Für Andrea")
- `hideCelebrant` — wenn die Reise kein Geburtstagskind hat, fällt die
  Stufe komplett weg (2-Stufen-Wahl)

**`PhotoShareSection`** (`src/components/photos/PhotoShareSection.tsx`)
- In PhotoDetail eingebettet — User tippt auf eigenes Foto → sieht
  Sharing-Optionen
- Zeigt aktuellen Sharing-Status (Online-Badge wenn schon geteilt)
- VisibilitySelector + „Jetzt teilen"/„Sichtbarkeit aktualisieren"/
  „Zurückziehen"-Button (kontextabhängig)
- Triggert SharingConsentModal automatisch wenn Einwilligung fehlt

**`SharedGallery`** (`src/components/photos/SharedGallery.tsx`)
- Neuer Tab "Gemeinsame Galerie" im Fotos-Bereich (oberhalb der eigenen
  Fotos)
- Zeigt alle vom Backend zurückgegebenen Fotos (server-side gefiltert
  via `canViewSharedPhoto`)
- 3-Spalten-Grid wie eigene Fotos, mit:
  - **Uploader-Badge** unten links (eigene Fotos mit ★ + Goldhintergrund)
  - **Visibility-Badge** oben rechts (🌐 Users-Icon / 🎂 Heart-Icon /
    🔒 Lock-Icon)
  - **⋯-Menü** oben links bei eigenen Fotos: Sichtbarkeit ändern oder
    zurückziehen
- **Andreas Spezial-Sicht**: zwei Tabs („Alle für mich sichtbaren" + „Nur
  für mich"), erscheint NUR wenn der Viewer das Geburtstagskind ist
- Empty-State wenn niemand was geteilt hat
- Service-not-configured-State (503-Fallback) wenn Vercel Blob/KV fehlt

### Neuer Hook

**`useSharedPhotos`** (`src/hooks/useSharedPhotos.ts`)
- Lädt geteilte Fotos für aktuelle Viewer-Identität via GET-API
- Actions: `share(args)`, `changeVisibility(id, level)`, `withdraw(id)`
- Auto-Refresh nach jedem Schreibvorgang
- `serviceConfigured: false`-State für 503-Antworten
- Klar getrennt vom existierenden `usePhotos` (lokal IndexedDB) —
  SharedGallery und persönliche Galerie können unabhängig versagen

### Integration in bestehende Flows

- `FotosTab`: nimmt jetzt `currentUserName` als Prop, rendert SharedGallery
  oberhalb der "Deine eigenen Fotos"-Sektion
- `PhotoDetail`: bekommt PhotoShareSection eingeklinkt zwischen Action-Row
  und AI-Narrator — alle Personen mit Avatar können von dort teilen
- `TripPageClient`: reicht `currentUserName` an FotosTab durch

## Was funktioniert NICHT (in dieser Version)

- **Aktivierung bei Vercel**: Storage-Vars müssen erst im Dashboard
  verbunden werden — bis dahin zeigt UI „Service wird eingerichtet"
- **Eigene-Foto-Indikator in Hauptgalerie**: noch nicht angezeigt ob
  ein eigenes Foto bereits geteilt wurde (außer wenn man's tippt und
  PhotoDetail öffnet). Könnte als Badge auf PhotoCard kommen.
- **Push-Notifications**: bei neuem geteiltem Foto kommt keine Mail/
  Notification. Refresh ist Pull-basiert beim Tab-Aufrufen.
- **Sortierung in SharedGallery**: aktuell nur chronologisch nach
  `takenAt` — keine Filter nach Uploader / Tag.
- **Caption-Editing nach Upload**: Caption-Änderung im PhotoDetail
  schreibt nur ins IndexedDB-Backup, nicht ins SharedPhoto-Record.
  Kann später nachgezogen werden.
- **Foto-Buch-Export inkludiert NICHT** die SharedPhotos der anderen.
  Aktuell läuft Export nur über lokale Fotos. Für Andreas Geburtstagsbuch
  müsste Foto-Buch-Export erweitert werden — kommt in v1.2.0.

## Migrations-Hinweise

### Vercel Storage Aktivierung (siehe auch v1.0.1)
1. Vercel Dashboard → Storage → Connect Blob (Region Europe)
2. Vercel Dashboard → Storage → Connect KV (oder Upstash for Redis)
3. Env-Vars erscheinen automatisch im Projekt
4. Redeploy ohne Cache

### Erste User-Story zum Testen
1. Andrea öffnet Foto-Tab → sieht „Service wird eingerichtet" (vor
   Aktivierung) oder leere „Gemeinsame Galerie" (nach Aktivierung)
2. Harald öffnet eines seiner Fotos → scrollt zur „Mit anderen teilen"-
   Sektion → wählt 🎂 → tippt „Jetzt teilen" → ConsentModal erscheint
3. Harald akzeptiert beide Checkboxen → Foto wird hochgeladen
4. Andrea refreshed ihren Foto-Tab → sieht Haralds Foto in „Nur für mich"
5. Andrea kann nicht widerrufen (es ist Haralds Foto), sieht ★ neben
   Haralds Namen wenn sie eigenes hochlädt

## User-Kommunikation

Sobald Vercel Storage aktiviert ist, WhatsApp-Nachricht an die
Reisegruppe:

> Update für die App: ihr könnt jetzt einzelne Fotos mit dem
> Geburtstagskind oder der ganzen Gruppe teilen. Default ist privat —
> jedes Foto entscheidet IHR selber. Beim ersten Teilen kommt eine
> einmalige Frage zu AGB+Datenschutz. Andrea bekommt dann eine extra
> Galerie „Nur für mich" mit allen Fotos die ihr für sie freigegeben
> habt. Ist alles freiwillig pro Foto, ihr könnt jederzeit zurückziehen.

## Datenfluss (Übersicht)

```
   USER hat Foto in IndexedDB (lokal)
              │
              ▼
   PhotoDetail-Tap → "Mit anderen teilen"
              │
              ├── 🔒 Privat       → bleibt lokal, nix passiert online
              │
              ├── 🎂 Für Andrea   → ConsentModal (wenn nötig)
              │                  → POST /api/photos/share visibility=celebrant
              │                  → Vercel Blob + KV
              │
              └── 🌐 Ganze Gruppe → ConsentModal (wenn nötig)
                                 → POST /api/photos/share visibility=group
                                 → Vercel Blob + KV

   GET /api/photos/list/london-2026?viewerName=Andrea
              │
              ▼
   server-side: canViewSharedPhoto(p, "Andrea", isCelebrant=true)
              │
              ▼
   nur die für Andrea sichtbaren Fotos kommen zurück
              │
              ▼
   SharedGallery rendert das Grid
```
