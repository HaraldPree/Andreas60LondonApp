# v1.13.0 — Leaflet interaktive Karte (Phase 2 aus PLAN.md endlich da)

**Datum**: 25.05.2026
**Status**: deployed
**Git Tag**: v1.13.0

## Anlass

Aus PLAN.md ursprünglich als „Phase 2" geplant, aus Lastenheft 7.9 explizit
gefordert, aus Polarsteps/Wayli-Wettbewerbs-Analyse als Lücke identifiziert:
**interaktive Karte mit POI-Markern, Filter und Geolocation**.

Bisher zeigte der Karte-Tab nur einen Link „in OpenStreetMap öffnen" + eine
Listen-Ansicht. Jetzt: echte Karte direkt im Tab.

## Was ist neu

### Neue Komponente `src/components/map/TripMap.tsx`

- **Leaflet + React-Leaflet 4.x** + OpenStreetMap-Tiles (kostenlos, kein API-Key)
- **POI-Marker** mit Kategorie-Farben (sight=navy, food=gold, transport=info,
  accommodation=success, hidden=warning) — Brand-konform
- **DivIcon-basiert** (HTML statt PNG-Assets) → keine Webpack-Asset-Issues mit
  Next.js, voll customizable
- **Popups** mit Name, Tag-Info, Direkt-Link zu Google Maps
- **User-Position-Marker** mit blauem Pulse-Ring wenn Geolocation aktiv
- **Auto-Fit-Bounds** zoomt automatisch so dass alle gefilterten Points + User
  sichtbar sind
- **`scrollWheelZoom={false}`** als Mobile-Default — verhindert Scroll-Klau
  beim Page-Scroll

### Integration im KarteTab

- 380 px hohe Karten-Card mit `rounded-2xl` + Brand-Border
- Kleiner „Groß"-Button oben rechts für OSM-Fullscreen-View (alte Funktion bleibt)
- Reagiert auf alle bestehenden Filter (Alle / In der Nähe / Tag X / Hidden) —
  Auto-Fit-Bounds passt sich live an
- Reagiert auf Geolocation — wenn User „Wo bin ich?" aktiviert, erscheint
  Marker + Karte zoomt entsprechend

### SSR-Lösung

Leaflet greift beim Mount auf `window` zu → SSR würde crashen. Lösung:

```tsx
const TripMap = dynamic(
  () => import("@/components/map/TripMap").then((m) => m.TripMap),
  { ssr: false, loading: () => <SkeletonMap /> },
);
```

Loading-Skeleton in Cream-Farbe, animate-pulse, während Tiles laden.

## Was funktioniert

- ✅ Alle POIs (Sights, Food, Hidden, Accommodation) als Marker sichtbar
- ✅ Filter (Tag 1/2/.../Hidden/Nearby/Alle) ändert Marker live
- ✅ User-Position als Pulse-Marker bei aktiver Geolocation
- ✅ Auto-Zoom auf gefilterte Punkte
- ✅ Popup mit Google-Maps-Link für Navigation
- ✅ Mobile-optimiert (Pinch-Zoom, Pan, Touch — alles Standard-Leaflet)
- ✅ OSM-Tiles laden weltweit, kein API-Key, kostenlos

## Was funktioniert NICHT (bekannte Limits)

- **Keine Tag-Linien zwischen POIs** — Tour-Routen-Anzeige käme als v1.13.x später
- **Keine Cluster-Marker** bei vielen POIs — wenn 100+ Markers nah beieinander,
  wird's unübersichtlich. `react-leaflet-cluster` als Folgevergröberung möglich
- **Kein Auto-Routing** zwischen Items (Wegfindung) — OSRM-Integration in v1.14+
- **Offline-Tiles**: aktuell brauchen Tiles Netzwerk. Service-Worker-Caching
  käme später
- **Bundle-Size**: +0,4 kB First-Load (sehr akzeptabel dank dynamic-import)

## Architektur-Notizen

- TripMap akzeptiert `focusPoint`-Prop (zukünftig: Klick auf Filter-Pill →
  Karte zoomt auf POI). Heute nicht verwendet, aber Architektur-vorbereitet.
- Marker-Farben aus `CATEGORY_COLORS_HEX` Konstante — wenn Tailwind-Theme
  geändert wird, hier mit-anpassen
- `useMap()` Hook in `ViewController`-Sub-Komponente für saubere Bounds-Logic
