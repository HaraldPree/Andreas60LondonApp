## Was ist neu

Erster Schritt der **Travel-Live-Vision**: ein neuer Tab „Wünsche" mit
einer kuratierten Sammlung von ~75 London-Sehenswürdigkeiten,
Geheim-Tipps, Filmspots und Foodie-Highlights. Pro Person markierbar
welche man **noch sehen will** (💭) oder **schon erledigt** (✓) hat.

### Pro Place: 3-Status-Toggle + Detail-Ausklapp

- **○ Offen** (Default, kein Eintrag)
- **💭 Auf meiner Liste** (Heart-Icon, gold)
- **✓ Erledigt** (Check-Icon, grün, durchgestrichen)

Tap auf den Body → Details:
- Vollständige Beschreibung
- Öffnungszeiten (Wochentage + Stunden)
- Quelle + letztes Verifikations-Datum
- Filmkontext bei Filmspots
- Karte + offizielle Website

### Live-Validierung gegen den aktuellen Wochentag

Wenn ein Place laut Öffnungszeiten **am heutigen Tag geschlossen** ist,
erscheint ein Warn-Pill „heute zu". Beispiel: Barbican Conservatory
(nur Sonntag) zeigt am Mi-Do das Warning, am So nicht.

```
Sir John Soane's Museum                   ○ ✓
Privat-Sammlung eines Architekten...      
[gratis] [🎨 Film] [heute zu]   ← Pill bei heute=Di
```

### Anti-Halluzinations-Disziplin

Jeder Place hat ein **`lastVerified: "2026-05-20"`** + **`source: "..."`**
Feld. Öffnungszeiten sind nach bestem Wissen erfasst, mit Verweis auf
offizielle Website. **Saisonale Schwankungen / Feiertage können
abweichen** — bei kritischen Besuchen offizielle Site selbst prüfen.

Konkretes Beispiel der „Alice-Korrektur" aus Notting Hill ist bereits
eingeflossen: **Barbican Conservatory** ist mit `openDays: ["So"]`
markiert (NICHT Mi/Do wie im alten Programm-Eintrag falsch stand).

### localStorage-Schema

Key: `rcmk:placeStatus:{tripSlug}:{userName}`
Value: `{ [placeId]: "wantToSee" | "done" }` (open wird nicht persistiert)

Beispiel:
```json
{
  "cedric-grolet-berkeley": "done",
  "tower-of-london": "wantToSee",
  "sky-garden": "wantToSee"
}
```

## Was funktioniert NICHT (in dieser Version)

- **Pro Gerät** — keine Gruppen-Synchronisation. Andrea sieht ihre
  Liste, Harald seine. Das **„was wollen wir gemeinsam noch sehen"**
  kommt in v1.7.1 mit Blob-Backend-Sync analog Foto-Sharing.
- **Live-Status-Layer** (Alice-Pattern) für Beobachtungen vor Ort —
  kommt ebenfalls in v1.7.1 (Read+Write für alle).
- **Cross-Trip-Bucket** — Places sind aktuell pro Trip definiert. Travel
  Live Phase C macht's lifelong.
- **AI-Plan-Generator** auf Basis der Bucket — Travel Live Phase B.
- **Bottom-Nav 7 Tabs** — Apple HIG empfiehlt max 5. Konsolidierung
  (SOS / Resv. in Info verlegen) kommt in einem späteren UX-Refactor.
